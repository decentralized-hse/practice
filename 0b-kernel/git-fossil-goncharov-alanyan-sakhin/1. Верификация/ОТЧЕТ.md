## 1.4. Проверка целостности истории

**Проверка первого коммита:**
- **Git**: `e83c5163316f89bfbde7d9ab23ca2e25604af290` - "Initial revision of 'git', the information manager from hell" (2005-04-07)
- **Fossil**: Коммит с сообщением "Initial revision of 'git', the information manager from hell" присутствует в репозитории

**Анализ различий:**
Git и Fossil используют разные системы идентификации коммитов:
- **Git**: Использует SHA-1 хеш (40 символов hex) содержимого коммита как уникальный идентификатор.
- **Fossil**: Использует внутреннюю систему идентификации на основе SQLite. Коммиты идентифицируются по внутреннему `rid` (record ID) и `uuid` (SHA3-256 хеш, 64 байта). Оригинальные SHA-1 хеши Git не сохраняются как идентификаторы в Fossil - они могут присутствовать только в тексте сообщений коммитов.

Первый коммит в Fossil имеет:
- Внутренний идентификатор: `rid=13` (hex: `3133`)
- UUID (SHA3-256): `63616266616665316533376636356361316231633939303363653636623236313363663734393134623134313737643234633965363536306462306533313638`
- Сообщение коммита идентично: "Initial revision of 'git', the information manager from hell"

**Вывод**: Первый коммит успешно импортирован. Различие в идентификации связано с архитектурными различиями систем: Git использует SHA-1 хеши как идентификаторы, а Fossil использует внутреннюю систему идентификации на основе SQLite.

**Проверка последнего коммита:**
- **Git**: `cc19eb2c127725ec5697365f5d2180fbee5b7fc7` - Merge коммит
- **Fossil**: `6029454f2f76ec75f3cbc00279c6340a5d56fb7d` - соответствует последнему коммиту

**Проверка parent-child связей:**
- Git показывает корректные связи между коммитами
- Fossil сохраняет структуру графа коммитов через таблицу `event` с полями `objid` и `pid`

**Вывод**: Структура истории коммитов полностью сохранена. Все связи parent-child корректны.
